Principal engineer: 6-month roadmap

4 hours/day · Project-driven · Distributed computing + ML specialisations

Phase 1 — Fortress of fundamentals
Months 1–2 · The bedrock everything else stands on
Foundation
Exit criteria (what you can do by end of month 2)
Implement any sort from scratch Explain Big-O intuitively Write a mini allocator in C Solve Leetcode mediums in Python Explain TCP/IP handshake precisely Prove basic linear algebra facts
Core topics
CS fundamentals How computers actually work: bits, memory, CPU, cache hierarchy, OS basics (processes, threads, virtual memory, syscalls)
Mathematics Discrete math (sets, logic, proofs), modular arithmetic, probability basics, linear algebra (vectors, matrices, dot product), calculus intuition
Programming fundamentals Memory model, pointers, stack vs heap, compilation pipeline, type systems, recursion, higher-order functions. Primary language: Python. Secondary: C basics.
DSA — Part 1 Arrays, linked lists, stacks, queues, hash maps from scratch. Big-O analysis. Binary search. Two-pointer & sliding window patterns.
Networking basics TCP/IP, DNS, HTTP/1.1 vs 2 vs 3, sockets, TLS handshake. Build a raw HTTP server with sockets.
Storage fundamentals File systems, B-Trees, how SQLite works internally, ACID properties, indexes. Read the SQLite design doc.
Projects (ship these)
1
Memory allocator in C
Implement malloc/free using sbrk(). Handle fragmentation, coalescing. Forces deep understanding of the heap, pointers, and OS memory management.
2
HTTP/1.1 server from raw sockets
Python or C. Parse raw TCP bytes, handle GET/POST, serve static files, support keep-alive. No frameworks. Teaches networking, byte parsing, concurrency primitives.
3
DSA library
Implement from scratch: dynamic array, linked list, hash map (with chaining + open addressing), binary search tree, min-heap. Write tests. Profile with cProfile.
4
25 Leetcode problems
Easy→Medium progression. Focus on arrays, strings, hashing. Blind 75 list. Write complexity analysis for every solution.
Key resources
CS:APP (Bryant & O'Hallaron) Nand2Tetris (Part 1) CLRS Ch 1–11 3Blue1Brown Linear Algebra Beej's Guide to Network Programming SQLite internals blog Python Fluent Python Ch 1–10
Phase 2 — Systems thinking + advanced DSA
Months 2.5–3.5 · Algorithms meet real systems
Intermediate
Exit criteria
Design a key-value store Explain CAP theorem with examples Solve graph problems (BFS/DFS/Dijkstra) Implement a basic DB query planner Profile + fix a perf bottleneck Understand latency numbers by heart
Core topics
DSA — Part 2 Trees (AVL, Red-Black), graphs (BFS, DFS, Dijkstra, Bellman-Ford, Topological sort), dynamic programming, greedy algorithms, tries, segment trees
Systems design fundamentals CAP theorem, consistency models, replication, partitioning/sharding, load balancing, caching layers (Redis internals), CDN, message queues (Kafka basics)
Performance engineering — foundations CPU cache effects, branch prediction, false sharing, SIMD basics, latency numbers (L1=1ns, RAM=100ns, disk=1ms). Profiling with perf, flamegraphs, py-spy.
Database internals B+ Tree vs LSM Tree, WAL (write-ahead log), MVCC, query planning, index design. Read CMU 15-445 notes. Implement a toy query planner.
Concurrency Threads, mutexes, semaphores, condition variables, deadlocks, lock-free data structures, memory ordering (happens-before). Python threading + asyncio deeply.
Platform basics Linux internals: scheduler, cgroups, namespaces. Docker from scratch. Understand how containers differ from VMs at the kernel level.
Projects (ship these)
1
Redis-lite: in-memory key-value store
Implement GET/SET/EXPIRE/TTL, persistence (RDB snapshot), pub-sub. Use Python asyncio for the event loop. Then benchmark it against real Redis.
2
LSM-Tree storage engine
Implement memtable → SSTable flush → compaction. This is the core of LevelDB/RocksDB. Forces deep understanding of write-optimised storage.
3
Performance profiling deep-dive
Take a slow Python program, generate flamegraph with py-spy, identify bottleneck, rewrite hot path in Cython or C extension. Measure before/after with benchmarks.
4
35 more Leetcode (medium/hard)
Graphs, DP, trees. Neetcode 150 list. Time yourself. Aim for 25 min per medium. Keep a "patterns notebook".
Key resources
DDIA (Designing Data-Intensive Apps) CMU 15-445 lecture notes CLRS Ch 22–25 (Graphs) Systems Performance – Brendan Gregg LevelDB internals blog Neetcode.io
Phase 3 — Distributed computing + ML specialisations
Months 4–5 · Two parallel tracks, 2 hours each per day
Specialisation
Exit criteria
Implement Raft consensus Build + train a neural net from scratch Explain Paxos vs Raft tradeoffs Deploy a distributed KV store Implement backpropagation by hand Profile distributed bottlenecks
Track A: Distributed computing
  • Consensus: Paxos, Raft, Zab
  • Distributed transactions: 2PC, Saga
  • CRDTs and eventual consistency
  • Vector clocks, Lamport timestamps
  • MapReduce, Spark internals
  • gRPC, Protobuf, service mesh
  • Kubernetes internals (scheduler, etcd)
  • Distributed tracing (Jaeger/OTEL)
  • Chaos engineering principles
Track B: Machine learning
  • Linear/logistic regression (derive it)
  • Gradient descent from scratch (numpy)
  • Neural nets: forward + backprop by hand
  • CNNs and weight initialisation
  • Transformers: attention mechanism
  • Optimisers: SGD, Adam, AdamW
  • Regularisation, batch norm, dropout
  • ML systems: data pipelines, feature stores
  • Model serving + inference optimisation
Performance + platform engineering (woven in throughout)
Performance engineering USE method (utilisation, saturation, errors), RED method, CPU/memory/IO profiling, tail latency analysis (p99, p999), queueing theory basics, Little's Law
Platform engineering IaC (Terraform), GitOps, CI/CD pipelines, observability stack (metrics + logs + traces), SLOs/SLAs/error budgets, on-call practices
Reliability engineering Failure modes, blast radius reduction, circuit breakers, bulkhead pattern, graceful degradation, load shedding, back-pressure
Security engineering mTLS, JWT internals, OAuth2/OIDC flows, secrets management (Vault), network policies, RBAC in Kubernetes
Projects (ship these)
1
Raft consensus implementation
Full Raft in Python or Go: leader election, log replication, snapshotting. Run 5-node cluster. Inject faults. This is the gold standard distributed systems project.
2
Neural network from scratch (numpy only)
MLP that trains on MNIST. Hand-implement forward pass, loss, backprop, weight updates. Then add batch norm. No PyTorch. Forces real understanding of gradients.
3
Distributed key-value store with Raft
Build on your Raft implementation. Add partitioning (consistent hashing), client retry logic, linearisable reads. Deploy on 3 cloud VMs.
4
Tiny transformer (character-level LLM)
Andrej Karpathy's nanoGPT, but every line explained. Train on Shakespeare. Then optimise inference: KV cache, quantisation basics.

Daily schedule — weekdays
4 hours of study + 10 hours of work, structured for deep focus

Morning block (primary)

5:30–6:00 Wake up. Water, no phone. Review yesterday's notes (15 min). Set today's single learning goal.
6:00–7:30 Deep study block 1. New concept or chapter. No distractions. This is your most cognitively fresh time — use it for the hardest material.
7:30–8:00 Exercise + shower. Non-negotiable. Physical state = cognitive state.
8:00–9:00 Deep study block 2. Coding problems or project work. Implement what you just learned.
9:00–10:00 Commute / prep for work. Listen to tech podcast or re-read notes.

Work day

10:00–1:00 Work (deep work block). Apply concepts from your study. Treat work projects as learning opportunities.
1:00–2:00 Lunch. Read a paper or blog post (15–20 min). Rest otherwise.
2:00–5:00 Work. During slow moments, write flashcards (Anki) on what you studied in the morning.
5:00–8:00 Dinner, family, downtime. Fully disconnect. Recovery is part of the system.

Evening block

8:00–9:00 Deep study block 3. Project work, coding, or re-reading. Light on new concepts — consolidation and building.
9:00–9:30 Anki review (spaced repetition). Write in your learning journal: what did I learn today, what's still unclear?
9:30–10:00 Wind down. No screens after 10pm. Plan tomorrow's goal.

Weekend study plan
6–7 hours Saturday · 4–5 hours Sunday · Rest is mandatory

Saturday — heavy learning day

6:00–6:30 Wake. Review week's notes. Identify the one gap from the week.
6:30–9:00 Deep block 1. New chapter or paper. Take handwritten notes. No skimming — go slow and understand deeply.
9:00–9:30 Break. Walk. Breakfast.
9:30–12:00 Project build session. Push your current project forward. Aim for a working end-state by day's end.
12:00–2:00 Lunch + rest. Nap if needed. Walk outside.
2:00–4:00 DSA practice. 3–4 Leetcode problems. Focus on patterns, not grinding. Analyse solutions you got wrong.
4:00–5:00 Write: blog post, or document what you built. Teaching = learning. Even a private doc works.
5:00 onwards Personal time. Full rest. Social. Do not study.

Sunday — consolidation day

7:00–7:30 Wake later. No alarm. Breakfast + review the week's learning journal.
8:00–10:00 Weak spot session. Whatever was hardest this week — revisit it. Read a different explanation. Draw diagrams. Rubber-duck it.
10:00–11:00 Anki review (full weekly batch). System design practice: sketch one system design from scratch in 45 min.
11:00–12:00 Plan next week: topics, projects, problems. Write 3 specific learning goals. Prep materials so weekday mornings start immediately.
12:00–2:00 Optional deep work. Only if energised. Project refinement, code review of your own work, or reading a research paper.
2:00 onwards Complete rest. Prepare mentally for the week. Sleep by 10pm Sunday.
